ntop Error Codes

  1. Wrong network mask or bridging enabled
    ntop will report this problem for those hosts that apparently have a wrong netmask. ntop detects this problem when a host sends a packet to a broadcast address where the destination MAC address is not FF:FF:FF:FF:FF:FF. This could indicate that:
    • such host has a wrong netmask
    • it is a bridge.
    Note that these warnings can be produced in case a mirrored traffic source is used and the -o flag has not been used.

  2. Duplicated MAC found for this IP address
    ntop will report this problem for those hosts that for a single MAC address sport several IP addresses. This could indicate that:
    • Someone is sending spoofed packets
    • A host has a wrong netmask hence it routes packets to the default gateway that routes them to destination. This is quite common on networks where the netmask has changed and some of the hosts still use the old netmask.
    Note that these warnings can be produced in case a mirrored traffic source is used and the -o flag has not been used.

  3. Traffic on suspicious IP ports
    ntop will report this problem for those hosts that produced traffic on ports where we should see no traffic (e.g. port zero).

  4. Suspicious activities: too many host contacts
    Usually a host contacts a limited number of hosts in a short amount of time. Some hosts (e.g. DNS/SMTP servers) can violate this rule but still behave normally. Other hosts that contact too many different hosts may be either attacked or infected by a virus that attempts to spread around. In this case you should carefully analyze the host in order to see whether this could really be an issue. Note that hosts with P2P applications, often contact several hosts, so this situation does not necessarely indicate a problem.
